# Графическая оболочка для анализатора C++ кода

## Описание

Графическая оболочка (GUI) для анализатора исходного кода C++, разработанная для работы в среде Astra Linux. Интерфейс создан на базе Tkinter, встроенного в Python, что обеспечивает совместимость и отсутствие необходимости в дополнительных зависимостях.

## Возможности

- **Управление конфигурацией**: Удобное редактирование всех параметров анализа через графический интерфейс
- **Запуск анализа**: Запуск анализа с выбором уровня контроля и метода поиска
- **Отслеживание прогресса**: Отображение прогресса выполнения анализа в реальном времени
- **Просмотр результатов**: Просмотр результатов анализа из базы данных PostgreSQL в удобном табличном виде
- **Экспорт отчетов**: Экспорт отчетов в текстовые файлы

## Требования

- Python 3.x
- Tkinter (обычно входит в состав Python)
- PostgreSQL и драйвер psycopg2
- Существующий анализатор C++ кода (все файлы из папки С++)

## Установка

1. Убедитесь, что все необходимые зависимости установлены:
```bash
sudo apt-get install python3-tk python3-psycopg2
```

2. Файлы GUI должны находиться в той же папке, что и анализатор (папка `С++`)

## Запуск

Запустите графическую оболочку командой:

```bash
cd С++
python3 gui_main.py
```

Или сделайте файл исполняемым:

```bash
chmod +x gui_main.py
./gui_main.py
```

## Использование

### Первый запуск

1. При первом запуске настройте параметры подключения к базе данных PostgreSQL
2. Укажите путь к проекту, который нужно анализировать
3. Настройте параметры анализа (уровень контроля, метод поиска)

### Запуск анализа

1. Выберите уровень контроля (2, 3 или 4)
2. Выберите метод поиска (ALL, FF, PFS и т.д.)
3. Нажмите кнопку "Запустить анализ"
4. Следите за прогрессом в окне "Журнал выполнения"

### Просмотр результатов

1. После завершения анализа нажмите кнопку "Результаты"
2. Просматривайте результаты во вкладках:
   - **Статистика**: Общая статистика по файлам, классам, функциям и переменным
   - **Файлы**: Список всех найденных файлов
   - **Классы**: Список найденных классов
   - **Функции**: Список найденных функций
   - **Переменные**: Список найденных переменных
   - **Связи между файлами**: Связи между файлами проекта
   - **Отчет**: Детальный отчет по результатам анализа

### Экспорт отчета

1. Выберите меню "Результаты" → "Экспорт отчета"
2. Укажите имя файла для сохранения
3. Отчет будет сохранен в текстовом формате

## Структура файлов

- `gui_main.py` - Главный файл приложения, создает интерфейс и координирует работу модулей
- `gui_config.py` - Управление конфигурацией анализатора (чтение/запись LOG_PATH.txt)
- `gui_analyzer.py` - Запуск анализа в отдельном потоке с отслеживанием прогресса
- `gui_results.py` - Просмотр результатов из базы данных и экспорт отчетов

## Параметры конфигурации

Все параметры хранятся в файле `LOG_PATH.txt` и могут быть изменены через интерфейс настроек:

- `DB_NAME` - Имя базы данных PostgreSQL
- `DB_USER` - Имя пользователя БД
- `DB_PASS` - Пароль пользователя БД
- `DB_HOST` - Адрес хоста БД
- `DB_PORT` - Порт БД
- `PATH_FILE` - Путь к анализируемому проекту
- `PATH_PYTHON` - Путь к исполняемому файлу Python
- `FILE_EXTENSION` - Расширения файлов для анализа
- `CHECK_DEPTH` - Уровень контроля (2, 3 или 4)
- `SEARH_METH` - Метод поиска (ALL, FF, PFS, C, F, V и т.д.)
- `DROP_TBL` - Сбрасывать ли таблицы перед анализом (0 или 1)

## Уровни контроля

- **Уровень 2**: Полный анализ (файлы, классы, функции, переменные, связи)
- **Уровень 3**: Анализ без некоторых дополнительных проверок
- **Уровень 4**: Минимальный анализ (только файлы и связи)

## Методы поиска

- `ALL` - Полный анализ
- `FF` - Поиск файлов
- `PFS` - Поиск путей и файлов
- `FF_PFS` - Комбинация FF и PFS
- `C` - Поиск классов
- `F` - Поиск функций
- `V` - Поиск переменных
- `CPI` - Проверка пакетов и импортов
- `ChI` - Проверка коммуникаций
- `CO` - Создание отчета
- `ND` - Проверка не обнаруженных функций
- `ADD_JSens` - Добавление JSensor

## Решение проблем

### Ошибка подключения к БД

- Проверьте настройки подключения к PostgreSQL
- Убедитесь, что PostgreSQL запущен
- Проверьте права доступа пользователя БД

### Ошибка "Python не найден"

- Укажите правильный путь к Python в настройках
- Или установите Python3: `sudo apt-get install python3`

### Ошибка "Модуль не найден"

- Убедитесь, что все файлы GUI находятся в папке с анализатором
- Проверьте наличие всех необходимых модулей Python

### Анализ не запускается

- Проверьте путь к проекту
- Убедитесь, что файл `_Starter.py` существует
- Проверьте права доступа к файлам проекта

## Поддержка

При возникновении проблем проверьте:
1. Журнал выполнения в окне GUI
2. Логи PostgreSQL
3. Права доступа к файлам и папкам
