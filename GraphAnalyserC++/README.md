# C++ Анализатор (Версия 2.0)

## 📋 Описание

Это **ТОЧНАЯ КОПИЯ** Python анализатора, адаптированная для C++.

Всё работает ТОЧНО ТАК ЖЕ, как в оригинале:
- Те же функции
- То же меню
- Та же логика создания деревьев и графов
- Та же запись в БД

**Префикс таблиц**: `cpp_` (вместо `ccc_`)

## 🚀 Быстрый старт

### 1. Настройка пути к проекту

Откройте `nodes_cpp.py` и измените путь:

```python
PATH_DIR = r"C:\путь\к\вашему\C++\проекту"
```

### 2. Запуск

```bash
cd test
python nodes_cpp.py
```

### 3. Выполните пункты по порядку (КАК В PYTHON!)

```
1. Найти все файлы C++           ← Сканирует проект
2. Найти все узлы и связи        ← Парсит файлы и сохраняет в БД
3. Создать деревья               ← Создаёт дерево каждого файла (PDF)
4. Заполнить граф                ← Создаёт объект граф (networkx)
5. Определить области видимости  ← ✅ ОБЯЗАТЕЛЬНО! (set_block_id)
6. Скрепить деревья              ← Опционально (связывает файлы через #include)
7. Заполнить граф + вложенность  ← ✅ ОБЯЗАТЕЛЬНО! (nesting_blocks)
8. Связать Call → FunctionDef    ← ✅ ОБЯЗАТЕЛЬНО! (find_func)
10. Создать граф                 ← Создаёт итоговый граф (PDF)
```

**ПОЛНАЯ последовательность: 1 → 2 → 4 → 5 → 7 → 8 → 10**

## 📊 Таблицы в БД

### `cpp_files`
- `id` - ID файла
- `path` - Путь к файлу
- `size` - Размер файла

### `cpp_type_nodes`
- `file_id` - ID файла
- `type_id` - ID узла
- `node` - **ИМЯ узла** (тип OR значение)
- `block_id` - ID блока видимости
- `lineno`, `col_offset` - Позиция в файле

### `cpp_parent_child`
- `file_parent_id`, `parent_id` - Родитель
- `file_child_id`, `child_id` - Ребёнок
- `field` - Тип связи (body, name, args и т.д.)

### `cpp_blocks`
- `file_id`, `block_id` - Идентификатор блока
- `file_node_id`, `node_id` - Родительский узел блока

### `cpp_nesting_blocks`
- `file_id_of_parent_block`, `parent_block_id` - Родительский блок
- `file_id_of_child_block`, `child_block_id` - Вложенный блок

### `cpp_func_def`
- `file_id`, `func_def_id` - Определение функции (FunctionDef)
- `func_name_id` - ID узла с именем функции
- `block_id` - ID блока видимости

### `cpp_func_calls`
- `file_id`, `call_id` - Вызов функции (Call)
- `call_name_id` - ID узла с именем вызова
- `attr_name` - Имя атрибута (если вызов через объект)
- `block_id` - ID блока видимости

### `cpp_func_calls_connections`
- `file_id_of_call`, `call_id` - Вызов функции
- `file_id_of_func_def`, `func_def_id` - Определение функции

## 🎯 Как это работает

### Парсинг C++ (find_nodes_cpp.py)

Файл `find_nodes_cpp.py` парсит C++ код и создаёт структуру **ТОЧНО КАК Python AST**:

```python
{
    'type': 'Module',              # Корневой узел
    'body': [                      # Тело модуля
        {
            'type': 'Import',      # #include директива
            'path': 'iostream',
            'names': ['iostream']
        },
        {
            'type': 'FunctionDef', # Определение функции
            'name': 'main',
            'args': {...},
            'body': [
                {
                    'type': 'Call', # Вызов функции
                    'func': 'printf'
                }
            ]
        }
    ]
}
```

### Создание графа (create_graph_cpp.py)

**ТОЧНО КАК В PYTHON**:
1. Загружает ВСЕ узлы из БД (без фильтрации)
2. Создаёт граф networkx
3. Связывает файлы через #include
4. Создаёт PDF
