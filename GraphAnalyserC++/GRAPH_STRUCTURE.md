# Структура графов граф-анализатора C++

## Что строится

Анализатор строит **два PDF** в каталоге вывода (например `graphs/`):

1. **graph.pdf** — полный граф: дерево AST + связи вызовов.
2. **graph_calls.pdf** — только граф вызовов: узлы «функция» и «вызов» и рёбра между ними (удобно для обзора архитектуры).

---

## Откуда берутся данные

| Этап | Таблицы/источник | Результат |
|------|-------------------|-----------|
| Сканирование | `cpp_files` | Список файлов (id, path). |
| Узлы AST | `cpp_type_nodes`, `cpp_parent_child` | Дерево: Module → body → FunctionDef/Call/Import/Namespace/ClassDef и т.д. |
| Области видимости | `set_block_id`, `cpp_blocks`, `cpp_nesting_blocks` | У каждого узла есть `block_id`. |
| Связи вызовов | `find_func_cpp` | В граф добавляются рёбра **Call → FunctionDef** при совпадении имени и области. |

Узлы AST заполняются из Clang (`find_nodes_clang_cpp`) или из таблиц ccc (определения/вызовы функций), либо минимально — по одному пустому Module на файл.

---

## Узлы в полном графе (graph.pdf)

- **Module** — корень по одному на файл (file_id).
- **FunctionDef** — определение функции/метода; подпись = имя функции (в т.ч. `Class::method`).
- **Call** — вызов функции; подпись = имя вызываемой функции.
- **Import** — включение заголовка (#include).
- **Namespace**, **ClassDef** — пространства имён и классы (без развёрнутого body в дереве).
- **body**, **name**, **func**, **args** — структурные рёбра и дочерние узлы (типы/имена).

Все блоки связаны: дерево родитель–потомок плюс дополнительные рёбра **Call → FunctionDef**.

---

## Рёбра

- **Серые** — структура дерева (родитель–потомок): body, name, func, args, индексы списков и т.д.
- **Синие** — вызовы: от узла **Call** к узлу **FunctionDef** (кто кого вызывает).

Подписи на рёбрах — имя поля связи (body, name, func и т.д.); у вызовов это обычно ключ ребра.

---

## Стиль и читаемость

- **Направление**: сверху вниз (TB).
- **Рёбра**: ортогональные (splines=ortho), чтобы связи не налезали.
- **Узлы**: цвет по типу (Module, FunctionDef, Call, Import, Namespace, ClassDef), скруглённые блоки для определений.
- **Подписи**: для FunctionDef и Call подставляется **имя функции** из дочернего узла (name/func), а не просто тип узла.

Граф **graph_calls.pdf** содержит только узлы FunctionDef и Call и синие рёбра вызовов — все блоки в нём связаны именно вызовами, без дерева AST.
